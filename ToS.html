1. Introduction

The ANON_PROTOCOL is a high-speed, ephemeral, web-based chat interface designed to facilitate instant, anonymous connections between two parties. Built on a serverless architecture utilizing real-time database synchronization, it prioritizes a "zero-persistence" philosophy where user identities are masked and session data is temporary.

Unlike traditional social media, there are no profiles, no long-term logs, and no central authority monitoring the nuance of your private conversation. It is a digital "dark room" for pure data exchange.

2. Technical Functionality

2.1 Peer Matching Logic

The protocol utilizes a Waiting Pool system. Upon initialization, the client generates a unique, session-based identifier (UID) stored in the local browser cache.

  • Queueing: If no other users are available, you are placed in a global waiting pool.
  • Handshake: When a second user joins, a "Room" is dynamically created. The two clients establish a synchronization link via a unique Room ID.
  • Anti-Reconnection Logic: To prevent harassment, the system implements a 15-second block on previous opponents if a session is terminated abruptly, ensuring you do not immediately rematch with the same person.

2.2 Communication Features

  • Textual Exchange: Real-time message streaming with sub-millisecond latency.
  • Multimedia Handling: Support for JPEG/PNG/WebP uploads. Images are automatically compressed and resized to 1024px to ensure stability and reduce bandwidth.
  • Spoiler Protection: Users can mark sensitive or "spoiler" images. These are delivered with a high-intensity Gaussian blur, requiring the recipient to click to reveal the content.
  • Contextual Replies: By clicking a message, users can "quote" a specific line of text to maintain clarity in fast-moving dialogues.
  • Presence Monitoring: Dynamic status indicators track whether the "Stranger" is Active, Away (window blurred), or Disconnected.

3. Operating Instructions

3.1 Starting a Session

Open the application in any modern web browser. The system will automatically attempt to "Connect." If the status dot remains orange, you are waiting for a partner. Once it turns green, communication is established.

3.2 Sending Content

Type in the bottom input bar and press Enter. To send an image, click the Square Icon. You may queue multiple images before sending. Click the "S" button on a preview thumbnail to toggle the Spoiler Blur before hitting Enter to send.

3.3 Managing the Connection

If at any point the conversation becomes undesirable, click the "Quit" button in the header. This will instantly destroy the room on the database level, preventing further messages from being sent or received.
